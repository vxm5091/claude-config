name: ticket-workflow-order
description: CEO/Eng reviews should happen before detailed ticket specs, not after
type: feedback
---

Reviews (CEO, Eng) should happen on high-level design, before writing detailed ticket descriptions with acceptance criteria.

**Why:** Detailed ticket specs written before reviews are wasted effort if the review changes scope, adds/removes tickets, or reshapes the work. The brainstorming phase should produce ticket names + one-liners, not full specs.

**How to apply:** When using the creating-linear-tickets workflow, brainstorming outputs a high-level design with rough ticket breakdown. CEO and Eng reviews refine scope. Only then write detailed ticket descriptions, acceptance criteria, and dependencies.
